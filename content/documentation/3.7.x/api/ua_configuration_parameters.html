Mandatory parameters h4(#parameter_uri). @uri@ SIP URI associated to the User Agent (@String@). This is a SIP address given to you by your provider. CODE_BEGIN "javascript" uri: "sip:alice@example.com" CODE_END h4(#parameter_sockets). @sockets@ Set of <%= my_lib_link_to "socket" %> instances. This parameter can be expressed in multiple ways: * Single <%= my_lib_link_to "socket" %> instance. * @Array@ of <%= my_lib_link_to "socket" %> instances. * @Array@ of @Objects@ defining a <%= my_lib_link_to "socket" %> instance with weight. Sockets with higher weight value are used prior to those with lower value. CODE_BEGIN "javascript" sockets: socket CODE_END CODE_BEGIN "javascript" sockets: [ socket1, socket2, ] CODE_END CODE_BEGIN "javascript" sockets: [ {socket: socket1, weight: 10}, {socket: socket2, weight: 1} ] CODE_END h2(#section_optional). Optional parameters h4(#parameter_authorization_jwt). @authorization_jwt@ JWT token (@String@) to use as authentication credential. If defined, every outgoing SIP request will contain a Authorization header using the Bearer scheme and the given value. CODE_BEGIN "javascript" authorization_jwt: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c" CODE_END h4(#parameter_authorization_user). @authorization_user@ Username (@String@) to use when generating authentication credentials. If not defined the value in @uri@ parameter is used. CODE_BEGIN "javascript" authorization_user: "alice123" CODE_END h4(#parameter_connection_recovery_max_interval). @connection_recovery_max_interval@ Maximum interval (@Number@) in seconds between WebSocket reconnection attemps. Default value is @30@. CODE_BEGIN "javascript" connection_recovery_max_interval: 60 CODE_END h4(#parameter_connection_recovery_min_interval). @connection_recovery_min_interval@ Minimum interval (@Number@) in seconds between WebSocket reconnection attempts. Default value is @2@. CODE_BEGIN "javascript" connection_recovery_min_interval: 4 CODE_END h4(#parameter_contact_uri). @contact_uri@ @String@ indicating the contact URI to be used in the Contact header field. The given URI host will be used as the Via header host parameter. h4(#parameter_display_name). @display_name@ Descriptive name (@String@) to be shown to the called party when calling or sending IM messages. It must NOT be enclosed between double quotes even if the given name contains multi-byte symbols (JsSIP will always enclose the @display_name@ value between double quotes). CODE_BEGIN "javascript" display_name: "Alice ¶€ĸøĸø" CODE_END h4(#instance_id). @instance_id@ @String@ indicating the UUID URI to be used as instance ID to identify the UA instance when using GRUU. CODE_BEGIN "javascript" instance_id: "uuid:8f1fa16a-1165-4a96-8341-785b1ef24f12" CODE_END CODE_BEGIN "javascript" instance_id: "8f1fa16a-1165-4a96-8341-785b1ef24f12" CODE_END h4(#parameter_no_answer_timeout). @no_answer_timeout@ Time (in seconds) (@Integer@) after which an incoming call is rejected if not answered. Default value is @60@. CODE_BEGIN "javascript" no_answer_timeout: 120 CODE_END h4(#parameter_session_timers). @session_timers@ Enable Session Timers (as per "RFC 4028":https://tools.ietf.org/html/rfc4028). Default value is @true@. CODE_BEGIN "javascript" session_timers: false CODE_END h4(#parameter_session_timers_refresh_method). @session_timers_refresh_method@ SIP method (@String@) to use for Session Timers refresh. Valid values are @UPDATE@ or @INVITE@. Default value is @UPDATE@. CODE_BEGIN "javascript" session_timers_refresh_method: 'invite' CODE_END h4(#parameter_session_timers_force_refresher). @session_timers_force_refresher@ Whether JsSIP shall force to be refresher of session timers for outgoing calls. When this is enabled JsSIP adds the "refresher=uac" parameter to the Session-Expires header in the initial INVITE. Default value is @false@. CODE_BEGIN "javascript" session_timers_force_refresher: true CODE_END h4(#parameter_password). @password@ SIP Authentication password (@String@). The plain SIP password. JsSIP deletes this value from its internal memory after the first successful authentication and, instead, stores the resulting @ha1@ and @realm@. CODE_BEGIN "javascript" password: "1234" CODE_END h4(#parameter_realm). @realm@ SIP Authentication realm (@String@). Just useful if plain SIP @password@ is not given, so it also requires @ha1@ to be provided. CODE_BEGIN "javascript" realm: "mydomain.com" CODE_END NOTE for Asterisk users: Asterisk always uses "asterisk" as @realm@. h4(#parameter_ha1). @ha1@ SIP Authentication HA1 hash (@String@) for the Digest authentication. If given (and plain SIP @password@ is not given) the parameter @realm@ is also required. Otherwise authentication is not feasible. CODE_BEGIN "javascript" ha1: "270fe19ce3890bd85e105998b0a75cf0" CODE_END h4(#parameter_register). @register@ Indicate if JsSIP User Agent should register automatically when starting. Valid values are @true@ and @false@ @(Boolean)@. Default value is @true@. CODE_BEGIN "javascript" register: false CODE_END h4(#parameter_register_expires). @register_expires@ Registration expiry time (in seconds) @(Integer)@. Default value is @600@. CODE_BEGIN "javascript" register_expires: 300 CODE_END h4(#parameter_registrar_server). @registrar_server@ Set the SIP registrar URI. Valid value is a SIP URI without username. Default value is @null@ which means that the registrar URI is taken from the @uri@ parameter (by removing the username). CODE_BEGIN "javascript" registrar_server: 'sip:registrar.mydomain.com' CODE_END h4(#parameter_use_preloaded_route). @use_preloaded_route@ If set to @true@ every SIP initial request sent by JsSIP includes a Route header with the SIP URI associated to the WebSocket server as value. Some SIP Outbound Proxies require such a header. Valid values are @true@ and @false@ @(Boolean)@. Default value is @false@. h4(#parameter_user_agent). @user_agent@ User-Agent header field value (@String@) present in SIP messages. By default "JsSIP" followed by the code version is set.
